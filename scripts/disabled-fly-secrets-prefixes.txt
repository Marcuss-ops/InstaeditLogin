# scripts/disabled-fly-secrets-prefixes.txt
#
# Single source of truth for the disabled-provider key prefixes. Sourced
# by:
#   - scripts/_parse_envfile.py   (rejects uncommented matches in the
#                                  .env file before push)
#   - scripts/verify-fly-secrets.sh (rejects matches in `flyctl secrets
#                                  list` output before declaring clean)
#
# Beta scope excludes these providers entirely — the set script refuses
# to push them, and the verify script refuses to declare clean if any
# of them are already registered. If you ever need to re-enable one of
# these, you must:
#   1. Remove the prefix from THIS file
#   2. Add the corresponding keys to scripts/required-fly-secrets.txt
#   3. Update docs/DEPLOY.md §3 (table + WHERE-to-get-it column)
#   4. Update the code paths in internal/services/ + pkg/api/ that gate
#      the provider behind capability flags (see internal/models/capabilities.go)
