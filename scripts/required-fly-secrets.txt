# scripts/required-fly-secrets.txt
#
# Single source of truth for the secrets we push to the `instaedit-login`
# Fly app. Sourced by:
#   - scripts/_parse_envfile.py   (loads the list, validates the .env file)
#   - scripts/verify-fly-secrets.sh (iterates to check `flyctl secrets list`)
#   - docs/DEPLOY.md §3           (the operator-facing table)
#
# Format rules:
#   - One key name per line
#   - Lines starting with `#` (after optional whitespace) are comments
#   - Blank lines are ignored
#   - Key names: uppercase ASCII + digits + underscore (POSIX env var name)
#
# Adding a new secret: add the key here, update docs/DEPLOY.md §3, update
# the rotation runbook if the key needs a special rotation dance, and
# add a CI test case in scripts/test_parse_envfile.py.
#
# Beta scope (27 secrets, per docs/DEPLOY.md + fly.toml secrets policy):
#   - 1 database, 3 crypto (JWT + ENCRYPTION_KEYS + ACTIVE id)
#   - 2 S3 (Tigris), 0 mail (Resend deferred until backend wiring)
#   - 2 Meta (App ID + Secret)
#   - 3 public-in-secrets (FRONTEND_URL, CORS, COOKIE_DOMAIN)
#   - 3 provider redirect URIs (Instagram, Facebook, Threads) — Meta-cognito
#   - 12 provider OAuth keys (X/TikTok/YouTube/LinkedIn × 3 each = CLIENT_ID + CLIENT_SECRET + REDIRECT_URI)
#
# NOTE: EMAIL_PROVIDER_KEY is intentionally NOT in this list. The backend
# does not yet wire Resend (magic-link / password-reset tokens are still
# returned in the response body as a dev fallback). Capture the key in the
# password manager for the Gmail inbox test, but do NOT push it to Fly
# until internal/services/email_sender.go is implemented.
#
# NOT INCLUDED (disabled providers — see
# scripts/disabled-fly-secrets-prefixes.txt for the guard list):
#   STRIPE_*
# (X_* / X_CLIENT_* moved to the required list above on X/Twitter
# enablement — see X_CLIENT_ID, X_CLIENT_SECRET, X_REDIRECT_URI.)
# (TIKTOK_* moved to the required list above on TikTok enablement —
# see TIKTOK_CLIENT_ID, TIKTOK_CLIENT_SECRET, TIKTOK_REDIRECT_URI.)
# (YOUTUBE_* moved to the required list above on YouTube enablement —
# see YOUTUBE_CLIENT_ID, YOUTUBE_CLIENT_SECRET, YOUTUBE_REDIRECT_URI.)
# (LINKEDIN_* moved to the required list above on LinkedIn enablement —
# see LINKEDIN_CLIENT_ID, LINKEDIN_CLIENT_SECRET, LINKEDIN_REDIRECT_URI.)
